# source-scripts

This case study was originally extracted from a private GTM repository. That production code (call-log API pulls, CVR enrichment, CRM sync, anonymisation) is not redistributable, and its anonymised source CSVs are not included here.

So the portfolio version is **fully self-contained and synthetic** — it takes no private input:

- `../python/build_dimensional_model.py` — a seeded synthetic generator. It designs a firmographic propensity model (employee band as the primary close signal, industry second, rep skill as the in-meeting lever, accounting system deliberately flat), auto-calibrates two knobs so the headline numbers reproduce on every run, and writes the fully fictionalised star schema to `../data/`.
- `../python/verify_numbers.py` — recomputes every quantitative claim in the README, deck, and dashboard spec from the generated CSVs. This is the canonical-numbers source of truth.
- `../python/analysis.py` — reproduces the five headline charts.
- `../sql/schema.sql` + `../sql/queries.sql` — DDL and 10 analytical queries, validated against the generated CSVs in DuckDB.
- `../powerbi/` — a working PBIP/TMDL Power BI project plus its spec, generated by `_gen_tmdl.py` / `_gen_report.py`.

Every figure in this case study is synthetic and deliberately round. It does not correspond to any real client's results — the point is to demonstrate the method (dimensional modelling, segment-lift analysis, scorecard design, BI delivery), not to report a real engagement.
